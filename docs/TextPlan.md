# 🚀 دليل تعديل كلاسات Tailwind لدعم العربي والإنجليزي (RTL & LTR)

القاعدة الذهبية: **انسى الاتجاهات الفيزيائية (يمين وشمال)، واستخدم الاتجاهات المنطقية (بداية ونهاية)**.

- **`s` (Start):** بداية الكلام (شمال في الإنجليزي / يمين في العربي).
- **`e` (End):** نهاية الكلام (يمين في الإنجليزي / شمال في العربي).

---

## 1. الهوامش الداخلية والخارجية (Margins & Paddings)

| ❌ متستخدمش ده (ثابت ويبوظ الصفحة) | ✅ استخدم ده (منطقي وبيقلب لوحده)                                               | الوصف                        |
| :--------------------------------- | :------------------------------------------------------------------------------ | :--------------------------- |
| `pl-*` (padding-left)              | **`ps-*` (padding-start)**                                                      | مسافة داخلية في بداية العنصر |
| `pr-*` (padding-right)             | **`pe-*` (padding-end)**                                                        | مسافة داخلية في نهاية العنصر |
| `ml-*` (margin-left)               | **`ms-*` (margin-start)**                                                       | مسافة خارجية في بداية العنصر |
| `mr-*` (margin-right)              | **`me-*` (margin-end)**                                                         | مسافة خارجية في نهاية العنصر |
| `space-x-*`                        | **`space-x-reverse`** _(مع تحديد الاتجاه)_ أو اعتمد على **`gap-*`** في الـ Flex | المسافة بين العناصر          |

---

## 2. المحاذاة وتحديد المواقع (Text Align & Positioning)

| ❌ متستخدمش ده | ✅ استخدم ده      | الوصف                                           |
| :------------- | :---------------- | :---------------------------------------------- |
| `text-left`    | **`text-start`**  | محاذاة النص لبداية السطر                        |
| `text-right`   | **`text-end`**    | محاذاة النص لنهاية السطر                        |
| `left-*`       | **`start-*`**     | تحديد موقع العنصر (Absolute / Fixed) من البداية |
| `right-*`      | **`end-*`**       | تحديد موقع العنصر (Absolute / Fixed) من النهاية |
| `float-left`   | **`float-start`** | تعويم العنصر لبداية الصفحة                      |
| `float-right`  | **`float-end`**   | تعويم العنصر لنهاية الصفحة                      |

---

## 3. الحدود والزوايا (Borders & Rounded Corners)

| ❌ متستخدمش ده                  | ✅ استخدم ده                        | الوصف                                 |
| :------------------------------ | :---------------------------------- | :------------------------------------ |
| `border-l-*`                    | **`border-s-*`**                    | حد (Border) في بداية العنصر فقط       |
| `border-r-*`                    | **`border-e-*`**                    | حد (Border) في نهاية العنصر فقط       |
| `rounded-l-*`                   | **`rounded-s-*`**                   | تدوير الزاويتين بتوع البداية          |
| `rounded-r-*`                   | **`rounded-e-*`**                   | تدوير الزاويتين بتوع النهاية          |
| `rounded-tl-*` / `rounded-tr-*` | **`rounded-ts-*` / `rounded-te-*`** | تدوير الزاوية العلوية (بداية / نهاية) |
| `rounded-bl-*` / `rounded-br-*` | **`rounded-bs-*` / `rounded-be-*`** | تدوير الزاوية السفلية (بداية / نهاية) |

## 7. حقول الإدخال الخاصة (Special Form Inputs)

مش كل حاجة في الفورم العربي بتبقى من اليمين للشمال! فيه حقول بطبيعتها "أجنبية"، لو سيبتها `rtl` المستخدم هيتعذب وهو بيكتب فيها (زي الإيميل لما يكتب فيه `@` أو الباسورد):

| نوع الحقل (Input Type) | الكود المطلوب إضافته | السبب |
| :--- | :--- | :--- |
| `type="email"` | **`dir="ltr" className="text-start"`** | الإيميل دايماً بيتكب بالحروف الأجنبية، لازم يبدأ من الشمال |
| `type="password"` | **`dir="ltr" className="text-start"`** | عشان الرموز والأرقام متهربش من المستخدم |
| `type="url"` (الروابط) | **`dir="ltr" className="text-start"`** | الروابط (https://) دايماً من الشمال |
| `type="tel"` (أرقام الهواتف) | **`dir="ltr" className="text-start"`** | عشان الكود الدولي (+20) يفضل في أول الرقم صح |

---

## 8. التدرجات اللونية والحركة (Gradients & Translate)

التدرجات اللونية والحركة الأفقية (زي فتح الـ Sidebar أو الـ Drawer) مش بتقلب لوحدها، لازم تعكسها بالـ `rtl:` :

| ❌ الكود الثابت | ✅ الكود المنطقي / العكسي | الوصف |
| :--- | :--- | :--- |
| `bg-gradient-to-r` | **`bg-gradient-to-r rtl:bg-gradient-to-l`** | عكس اتجاه تدرج الألوان في العربي |
| `bg-gradient-to-l` | **`bg-gradient-to-l rtl:bg-gradient-to-r`** | عكس اتجاه تدرج الألوان في العربي |
| `translate-x-full` | **`translate-x-full rtl:-translate-x-full`** | تحريك عنصر مخفي (زي القائمة الجانبية) عشان يفتح من الناحية الصح |
| `-translate-x-full` | **`-translate-x-full rtl:translate-x-full`** | إخفاء العنصر في الاتجاه العكسي |

---

## 9. الخطوط الفاصلة بين عناصر القوائم (Divide)

لو بتعمل قائمة أفقية وفصلت بين عناصرها بخط باستخدام `divide-x`، الخط ده هيبوظ في العربي وهتلاقيه مرسوم في الناحية الغلط:
- ❌ **متستخدمش:** `divide-x` لوحدها.
- ✅ **الصح:** ضف معاها **`divide-x-reverse`** (أو الأسهل والأحدث: استخدم `flex` مع **`gap-*`** واعمل الحدود بنفسك جوه العناصر).

---

## 10. إعدادات خطوط النظام (CSS Font Family Snippet)

انسخ الكود ده وحطه في أول ملف `index.css` أو `App.css` عشان تضمن إن الخطوط العربية والأجنبية متتداخلش:

```css
/* إجبار المتصفح على استخدام خط عربي متناسق في حالة الـ RTL */
html[dir="rtl"], html[dir="rtl"] * {
  font-family: 'Cairo', 'Almarai', 'Tahoma', sans-serif;
}

/* استخدام خط أجنبي مودرن في حالة الـ LTR */
html[dir="ltr"], html[dir="ltr"] * {
  font-family: 'Inter', 'Poppins', 'Segoe UI', sans-serif;
}
```tsx
// الصَّح
<ArrowRight className="w-5 h-5 rtl:rotate-180" />
<ChevronRight className="w-4 h-4 rtl:rotate-180" />
```
